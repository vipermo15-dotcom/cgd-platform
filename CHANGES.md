# 개선 작업 내역

CGD 취업지원 플랫폼 코드베이스 분석 후 진행한 보안·성능·정합성 개선 및 기능 보완 기록.

검증 기준선(매 단계): **`pnpm check` 0 errors · `pnpm test` 30/30 pass · `pnpm build` 성공**

---

## 1. 보안 — 역할 기반 접근 제어 중앙화
**문제**: `protectedProcedure`(인증만 확인)가 라우터 전반에 쓰여, 인증된 사용자가 타 역할 API를 직접 호출 가능. 역할 가드가 일부 라우터에 중복 정의돼 있었음.

**변경**:
- `server/_core/trpc.ts`: `requireRole()` 미들웨어 팩토리 + `studentProcedure`/`professorProcedure`/`companyProcedure`/`trainingProcedure` export. admin은 모든 역할 자동 우회.
- `professor.ts`, `company.ts`의 로컬 가드 정의 제거 → 중앙 import.
- `cgd.test.ts`: admin 우회·교차 역할 거부·미인증 거부 테스트 5건 추가.

## 2. 보안 — AI 프롬프트 인젝션 방어
**문제**: 사용자 입력(포트폴리오 제목·설명, 회사명, 직무)이 정제 없이 LLM 프롬프트에 삽입됨.

**변경**:
- `server/_core/promptSafety.ts` 신규: `sanitizeForPrompt`(제어문자 제거, 코드펜스·가짜 역할태그·`system:` 프리픽스 무력화, 길이 제한), `sanitizeList`, `wrapUntrusted`(구분자 래핑), `UNTRUSTED_DATA_NOTICE`.
- 적용: `ai.ts`(analyze·generateCoverLetter), `guidance.ts`(aiRecommendCompanies), `jobs.ts`·`resume.ts`(추천 로직).

## 3. 성능 — 프론트엔드 코드 분할(lazy loading)
**문제**: `App.tsx`가 30+ 페이지를 정적 import → 초기 번들에 모든 역할 코드 포함.

**변경**:
- Home/Login/NotFound만 즉시 로딩, 나머지 33개 페이지 `React.lazy` + `<Suspense>`.
- 빌드 시 역할/페이지 단위 청크 분리 확인(차트 라이브러리 포함).

## 4. 비용 — AI 엔드포인트 rate limiting
**문제**: LLM 호출 API에 호출 빈도 제한 없음.

**변경**:
- `server/_core/rateLimit.ts` 신규: `ai_logs` 기반 슬라이딩 윈도우(재시작·다중 인스턴스 안전). `db.ts`에 `countRecentAiLogs` 추가.
- 한도: 역량분석 10분/5회, 자기소개서 10분/10회, 취업처추천 10분/20회.
- `aiRecommendCompanies`에 누락됐던 ai_logs 기록 추가(카운팅·비용추적 겸용).

## 5. 성능 — DB 인덱스
**문제**: 빈번한 WHERE/JOIN 컬럼에 인덱스 없음(rate limiting이 조회하는 `ai_logs` 포함).

**변경**:
- `drizzle/schema.ts`: 15개 테이블 28개 인덱스 선언(userId/status/approvalStep/studentUserId 등).
- `drizzle/0007_fearless_synch.sql`: drizzle-kit으로 정식 마이그레이션 생성 + bookmarks 백필 UPDATE 추가.

## 6. 설계 — bookmarks 테이블 정리
**문제**: 채용공고/인재 북마크가 판별자 없이 한 테이블 혼용. `getUserBookmarks`가 학생 북마크까지 섞어 반환(버그).

**변경**:
- `type`(job/student) 판별자 컬럼 + 복합 인덱스 2개.
- `getUserBookmarks`를 `innerJoin` + `type='job'` 필터로 교정.
- toggle 함수에 type 명시, 마이그레이션에 기존 데이터 백필 포함.

## 7. 정합성 — 지원 중복 + applicantCount
**문제**: 중복 지원 가드 부재(반복 호출 시 행 중복 + 카운트 과다). 삭제 시 카운트 미감소로 드리프트.

**변경**:
- `createJobApplication`: `(jobPostingId, applicantUserId)` 중복 가드(null 반환), 카운트를 `COUNT(*)`로 재계산해 자가 치유.
- `submitApplication`: 중복 시 `CONFLICT` 에러로 사용자 안내.

## 8. 정리 — 타입 캐스팅 / 문서
- `ai.ts`의 불필요한 `response_format ... as any` 제거(`ResponseFormat` 타입에 이미 포함).
- `todo.md` 5차 항목을 실제 구현 상태와 교차 검증 후 현행화.

## 9. 기능 — 최종합격 시 취업 축하 배너 자동 생성
**변경**:
- `jobs.updateApplicationStatus`가 '최종합격' 시 `autoCreateEmploymentBanner` 호출.
- 중복 방지(학생+회사+isAutoGenerated), `student_profiles` 취업상태 '취업확정' 동시 갱신, 이니셜 처리 기본값.
- 회귀 가드 테스트 추가.

## 10. 기능 — 희망기업 매칭 페이지(`/student/job-matching`)
**변경**:
- 기존 엔드포인트만으로 구성: AI 매칭 결과(점수·사유) + 내 희망기업(북마크) + 지원 다이얼로그.
- 이미 지원한 공고는 '지원완료' 배지 표시. App.tsx lazy 라우트 + 사이드바 메뉴 추가.

## 11. 기능 — 채용공고 첨삭 (재학생 ↔ 멘토)
**변경**:
- 신규 테이블 `job_coaching_requests`(희망공고 + 첨부서류 + 첨삭결과/상태).
- 서버 `coaching` 라우터: 학생(요청 생성/조회/삭제) + 검토자(목록/상세/검토중/첨삭완료+알림).
- 학생 페이지 `/student/job-coaching`: 공고 입력(회사·직무·URL·내용) + 내 서류 첨부 + 요청 목록 + 첨삭결과(Markdown).
- 검토자 페이지 `/admin/job-coaching`(학과장·관리자·공동훈련센터): 요청 목록·상태 필터 + 상세(첨부서류 열람) + 첨삭 작성.
- 사이드바 메뉴 4개 추가, lazy 라우트, 마이그레이션 0008, 권한 테스트 3건.
